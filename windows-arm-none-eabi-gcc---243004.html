windows arm-none-eabi-gcc 交叉编译环境以及固件烧写功能配置

Posted on Wed, Aug 21, 2024 embedded

安装winget和gnu-make

winget 应该已经预装在windows11 内了，windows10 环境可以参考https://sspai.com/post/67005 这篇说明，我们需要用到winget安装gnu-make

winget安装完成后，打开powershell 输入 

winget install GnuWin32.Make

按照提示完成安装后重新启动powershell ，输入make命令测试是否正常。

安装arm-none-eabi-gcc

https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads 在这个页面，选择第一个下载安装

安装完成后打开powershell, 输入 arm-none-eabi-gcc -v，检查打印信息是否正确。

安装pyocd

pyocd 对python版本要求在3.6以上，安装命令 

pip install -U pyocd

pyocd 详细使用教程参考 http://www.51hei.com/bbs/dpj-194723-1.html

以stlink 下载器为例，stlink 驱动已正确安装，打开powershell 输入

pyocd list

应显示已连接的stlink 下载器信息

pyocd 下载固件需要下载对应型号单片机的pack包，可以使用pyocd pack install 命令，也可以在https://www.keil.com/dd2/pack/ 下载并手动加载。以stm32f407zg 为例

在线下载pack

pyocd pack install stm32f407zg

手动加载命令如下

pyocd load —pack Keil.STM32F4xx_DFP.2.17.0.pack

需要注意的是，如选用手动加载，则每次执行固件烧写或擦除命令都需要带上load 参数

pyocd load —pack Keil.STM32F4xx_DFP.2.17.0.pack flash -t stm32f407zg app.hex

固件烧写与擦除

以stm32f407zgt6 为例

烧写hex

pyocd flash -t stm32f407zg app.hex

烧写bin

pyocd flash -t stm32f407zg —base-address 0x8000000 app.bin

整片擦除

pyocd erase -t stm32f407zg —chip
